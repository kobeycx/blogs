概述

Swift 语言有两种基本的数据类型，即类(class)和结构体(struct). 在其它有 struct 的语言中，struct 一般只拿来组织逻辑相关的数据. 但在 Swift 中, struct 除此之外, 还拥有 init, method, 除了无法被继承外, 和 class 的使用方法几乎一样. 在实际使用中, 在不需要继承的情况下, 应该使用哪种?

区别

  • struct 值引用, 不能继承.
  • class 内存引用, 可以继承.

为什么会出现这种情况呢? 因为对于两者，处理的方式是不一样的.

class 为 引用类型, 也就是说, aClass 和 bClass 指向的是同一块内存区域

如果声明一个 class 为常量, 那么就意味着该常量的引用不能改变（即不能被同类型变量赋值, 但指向的内存中所存储的变量是可以改变的

struct 为 值类型, 也就是说, 每个实例都单独持有一份内存区域

如果声明一个 struct 为常量, 那就意味该常量是不可变的(无论内部数据为 var 或 let)
